# Security Policy

## Supported Versions

Security reports are accepted for the latest code on the default branch.

## Reporting a Vulnerability

Please report security issues privately through GitHub's private vulnerability reporting if it is enabled for the repository. If that is not available, contact the maintainer directly instead of opening a public issue.

Do not attach real crash reports, proprietary stack traces, or user-identifying logs to public issues.

## Data Handling

Crash reports can include sensitive metadata. The app is designed to run locally and does not upload reports to any external service. The repository ignores real `.ips` and `.crash` files by default.
