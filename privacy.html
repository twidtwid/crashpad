CrashPad

Privacy Policy

CrashPad is designed to analyze Apple crash reports in your browser. Reports you choose or drop are read locally in the current tab, not posted to this server, and are not stored by the application.

Check out the GitHub project if you would like to run this completely locally.

What The App Processes

When you choose or drop a crash report, the file is read by browser JavaScript in the current tab so the report can be parsed and rendered. The file picker and drop target are local browser inputs, not a network upload. The app keeps the parsed report in page memory only.

What The Server Stores

The server hosts the static app, read-only sample routes for the public QLThumbnail example report, and aggregate stats routes. It has no route that receives crash report files and never writes chosen reports to disk.

Aggregate Stats

CrashPad records non-identifiable aggregate stats such as visits, reports analyzed, example versus browser-local analyses, parse failures, and user-triggered print, export, and copy actions. The stats store is public counters only and does not store IP addresses, user agents, file names, report contents, stack traces, or identifiers.

Ephemeral Controls

Click Forget Report, reload the page, or close the tab to clear the current report from browser memory. Export JSON and Copy Summary are local browser actions you initiate.

Operational Logs

Hosting infrastructure may record ordinary request metadata such as IP address, URL path, user agent, and timestamps. Do not put sensitive crash data in URLs. Crash report file contents are not transmitted by the file picker or drop target.

Public Example

The built-in QLThumbnail example is a sanitized fixture intended for demonstration and testing.
